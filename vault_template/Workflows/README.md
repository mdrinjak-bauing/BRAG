# Workflows — deine wiederkehrenden Aufgaben

Kurze Vorlagen für Dinge, die immer gleich ablaufen. Du nennst eine beim Namen
(„hol mich auf den Stand", „halt das fest"), und Claude liest die passende Datei
(mit `read_note`) und führt die Schritte mit den BRAG-Werkzeugen aus.

**So kennt Claude sie:** In der `CLAUDE.md` (die du in die Projekt-Anweisungen
einfügst) steht je Workflow eine Auslöser-Zeile. Erst dadurch reagiert Claude auf das
Stichwort — die Dateien hier werden nicht von allein gelesen.

**Eigene anlegen:** neue `.md` hier ablegen — Titel = das Thema, dazu ein **Auslöser**
(welche Worte ihn starten) und nummerierte Schritte mit den Tools. Dann eine
Auslöser-Zeile in die `CLAUDE.md` ergänzen. Ideen: „Tagebucheintrag schreiben",
„Wochenrückblick erstellen", „Protokoll aus den Notizen ziehen".

(Liegt im WissensWIKI — wird **nicht** durchsucht/indexiert.)
